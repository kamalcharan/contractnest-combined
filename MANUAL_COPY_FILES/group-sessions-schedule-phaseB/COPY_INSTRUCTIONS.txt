WHAT THIS ADDS
----------------------------------------------------------------
ONE shared schedule per group-session block (not per member contract), so a
holiday reschedule is a single-row edit every member sees.

  New table t_group_session_schedule (tenant, source_block_id, is_live,
  occurrence_date, seq, status, note). Generated from the block cadence
  (m_cat_blocks.config.serviceCycles: days + anchorWeekday).

  New RPCs: gs_generate_schedule, gs_schedule_move (holiday), gs_schedule_status
  (skip/cancel/restore/held), gs_schedule_add, gs_renumber_schedule.
  gs_dash_occurrences now reads this table; gs_dash_sessions aggregates
  occurrences_total / done / next_occurrence from it.

Dashboard → open a group session → Occurrences tab:
  - empty  → "Generate schedule" button (from the block cadence)
  - filled → each occurrence with # / date / status; per-row actions:
             Move date (holiday), Skip, Cancel, Restore; plus "Add date"

Production checklist:
  Transaction Mgmt : single-statement upserts; renumber in one UPDATE
  Race conditions  : UNIQUE(tenant, block, is_live, date) — no dupe dates
  Error handling   : RPC {ok:false,reason}; service {success,error};
                     controller sendError; mutation onError toasts
  Toasts           : react-hot-toast on every schedule mutation
  Loaders          : buttons disable + "Generating…" while pending

TYPECHECK: api + ui group-sessions files compile clean (tsc --noEmit).
