APPLY AFTER: cadence-buyer-acceptance. SUBMODULE: contractnest-api only.

ROOT CAUSE (found on CN-1043): the VaNi composer's assembleDraft
hardcoded categoryId 'service' for EVERY block it instantiates from a
template. The auto-included Terms & Conditions text block therefore
arrived as a "service" block → the event derivation gave the ₹0 block
a Prepaid billing event → the events mapper turns 0 into null → the
null-amount day-1 billing task. Also mislabeled headings (text/billing
blocks shown as Service).

FIX: CandidatePayload += category_id; template candidates carry the
saved block category; catalog-scan candidates stay 'service' (that
tier only matches service blocks); assembleDraft + derivationBlocks
use the real category. The API derivation already skips non-priced
categories — it just had to be told the truth.

FILE: contractnest-api/src/services/contractComposerService.ts

# restart API server
