WHY
  You completed an onboarding and the Smart Profile never triggered — because it
  was never wired in. Onboarding ALREADY captures the website URL at the branding
  step (BusinessPreferencesStep) but did nothing with it. This makes that URL
  actually build the tenant's Smart Profile — in the background — so the ICP is
  ready to power resource-seeding rank (Piece 2) + smart chips + nomenclature.

WHAT IT DOES
  When the branding step is submitted WITH a website URL, the UI fires a
  fire-and-forget call to a new endpoint. The server responds 202 immediately and
  runs the SAME pipeline the Settings wizard uses, detached:
      scrape-website (n8n) → save profile → generate clusters (n8n) → save clusters
  Onboarding is NEVER blocked (no spinner, no wait). By the time the tenant
  reaches resource seeding (~2 steps later) the profile is usually ready; if not,
  seeding/chips degrade to archetype ranking.

  TWO HARD RULES (enforced server-side):
    • HONEST — if the scrape/AI returns no real content, it persists NOTHING. No
      fabricated description, no placeholder keywords. (Your standing rule.)
    • IDEMPOTENT — if the tenant already has a profile, it SKIPS. Never clobbers a
      hand-authored profile, never double-builds on a re-submit.
