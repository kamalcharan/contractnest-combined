PROBLEM
─────────────────────────────────────────────────────
All three screens failed to restore previously saved data
when the user returned mid-onboarding.

ROOT CAUSES
─────────────────────────────────────────────────────
Screen 3 — PersonaSelectionStep:
  fetchProfile() populates formData.business_type_id
  asynchronously, but there was no effect watching that
  field — selected state was only ever set by user clicks.

Screen 4 — ThemeSelectionStep:
  Read user?.preferred_theme from AuthContext, which may
  not be loaded at mount time. ThemeContext's currentThemeId
  (sourced from localStorage) is available synchronously and
  is the correct source of truth for the active theme.

Screen 5 — IndustrySelectionStep:
  useServedIndustries is enabled: !!currentTenant?.id.
  When currentTenant is null on mount (auth still loading),
  the query is DISABLED → TanStack Query returns isLoading=false
  immediately with empty data. The hydration effect ran, saw
  isLoading=false + empty data, set hydrated=true — then when
  the real fetch finally ran, hydrated=true blocked it.

FIXES
─────────────────────────────────────────────────────
PersonaSelectionStep.tsx:
  + useEffect watching formData.business_type_id
  + When formData arrives from fetchProfile(), finds matching
    PERSONA and sets selected + shows VaNi bubble immediately

ThemeSelectionStep.tsx:
  + Switched from user?.preferred_theme to currentThemeId
    (from useTheme() — always available synchronously from
    localStorage before setTheme('vani') overwrites it)
  + If pre-selected, VaNi reaction bubble shown immediately

IndustrySelectionStep.tsx:
  + Added currentTenant?.id guard to hydration effect
  + Effect now returns early if tenant not yet loaded
  + Correctly waits for query to be enabled before initializing
  + If pre-populated, VaNi reaction bubble shown immediately
