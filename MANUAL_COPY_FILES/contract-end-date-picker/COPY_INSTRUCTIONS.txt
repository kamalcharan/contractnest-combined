FEATURE: Exact End Date picker in Contract Details (bidirectional with duration)
DATE: 2026-07-13
BRANCH: claude/group-session-handover-reset-bsn220
═══════════════════════════════════════════════════

WHAT CHANGED:
─────────────────────
Added an "End Date" date picker beside "Start Date" in the Contract Details
wizard step. It stays in sync with the duration presets / duration input:
  - Change Start Date or duration  -> End Date updates automatically.
  - Pick an exact End Date          -> duration back-computes (whole years or
    months when the date lands cleanly on one, otherwise an exact day count).

WHY NO API / DB / EDGE CHANGE:
─────────────────────
end_date is already STORED in the DB, computed by the trigger
trg_compute_contract_dates as start_date + duration (migration
054_end_date_prolongation_auto_expire.sql). The trigger always derives
end_date from duration, so we keep duration as the source of truth and derive
it from the picked date. The stored end_date will exactly equal the date you
pick (day-count fallback is Postgres-exact; month/year only used when it maps
cleanly with no month-end overflow).
