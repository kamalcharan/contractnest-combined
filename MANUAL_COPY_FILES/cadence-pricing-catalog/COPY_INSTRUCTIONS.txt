WHAT THIS DELIVERS
In Catalog Studio → Block Wizard → Pricing step, each currency card
for a FIXED-price service gets a scheme choice:

 1. "One price" (default) — exactly today's behavior. Every existing
    block, and every block where you don't touch the toggle, works
    unchanged.
 2. "Varies by billing cadence" — reveals:
    a. The ANCHOR (captured & stored): "Total price ₹X for Y months".
       X reuses the record's price field; Y is new (base_term_months).
       Nothing assumes a 1-year timeline.
    b. The rate card: Monthly / Quarterly / 6 Months / Annual rows,
       each with an enable checkbox + per-period rate. Each enabled
       row shows the FAIR pro-rata value vs your anchor, a one-click
       "use" link to fill it, an at-par/+premium/−discount tag, and
       "= ₹N over the Y-month base term".
    c. A DEFAULT radio — one enabled cadence, pre-selected when the
       block is later added to a contract. Auto-moves if you disable
       the default row.

VALIDATION (Next is blocked with a message if):
  - cadence scheme with no base total or no base term
  - no enabled cadence with a rate > 0
  - no default among the enabled cadences

STORAGE / DATA MODEL (no DB migration)
  New optional fields ride inside each pricingRecord in the block's
  meta JSON (config), which already round-trips through
  catBlockAdapter untouched:
    pricing_scheme: 'single' | 'cadence'
    base_term_months: number
    cadence_rates: [{ cycle, amount, enabled }]
    default_cadence: 'monthly'|'quarterly'|'halfyearly'|'annual'
  Shared types/math live in the NEW module
  src/utils/catalog-studio/cadencePricing.ts — the wizard and buyer
  steps will import from the same place later.

INTERIM BEHAVIOR (until the wizard-side step lands)
  A cadence-priced block added to a contract TODAY still behaves as a
  single-price block using the anchor total (₹X) as its price — the
  contract wizard doesn't read the rate card yet. That's the next
  step, after this gate passes.

NOT SUPPORTED IN V1 (flag if you want them)
  - Cadence pricing on HOURLY price type (toggle hidden — per-period
    rates don't make sense against an hourly rate)
  - Cadence pricing inside PER-VARIANT pricing mode (the per-variant
    cards keep single prices; "Same for All" + cadence works)
