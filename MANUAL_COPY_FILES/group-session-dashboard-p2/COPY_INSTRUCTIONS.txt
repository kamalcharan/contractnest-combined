WHAT THIS IS
----------------------------------------------------------------
The generic (tenant-agnostic) Group Sessions dashboard UI, wired to the
Phase 1 gs_dash_* RPCs through new Express endpoints. Drill-down:
  Overview (session cards) -> Session (Occurrences | Roster tabs)
                           -> Member (attendance history + dues)
New nav item under Operations -> "Group Sessions" (/group-sessions).
"Open check-in" links to the existing chair flow (/session-checkin).

IMPORTANT: the UI never calls Supabase directly - all data goes through
the Express API. That's why this phase adds API endpoints too. tenant_id
and is_live are injected by the api request interceptor as headers
(x-tenant-id / x-environment); the backend reads them - nothing is
hardcoded and no tenant name appears anywhere.

TYPECHECK: both new UI files and the API files compile clean (tsc --noEmit).
(Note: a PRE-EXISTING broken file src/services/serviceURLs.addition.ts on
UI main throws tsc errors - it is unrelated to this work and left untouched.)
