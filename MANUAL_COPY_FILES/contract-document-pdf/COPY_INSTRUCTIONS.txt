WHAT THIS DELIVERS
  A shared ContractDocument component renders the professional paper
  (letterhead → reference grid → parties → Schedules A/B/C → terms →
  signatures → CNAK footer). It is rendered OFF-SCREEN and captured
  by the existing html2canvas+jsPDF flow, so:
   - Wizard Review step → "Download PDF" now produces the pro document
   - Buyer review page (/contract-review) → same
  The on-screen review pages are unchanged (their redesign can adopt
  the same component later if you want).

CONTENT SOURCES (all dynamic, per your correction)
  Letterhead + Provider  → tenant profile (name, logo, city, phone,
                           email); initials monogram when no logo
  Contract No / status   → contract fields ("Draft — number on save"
                           in the wizard before first save)
  Reference grid         → effective date, term → end date,
                           acceptance method, currency
  Customer               → buyer contact (salutation-aware name,
                           phone, email)
  Intro sentence         → GENERATED from provider/customer/term
  Schedule A             → service & spare blocks: service cycle →
                           "Every N days", visit counts; billing-only
                           blocks badged with no visits
  Schedule B             → per-block cadence/rate/payments/amounts +
                           the locked chain: Sum total → Discount →
                           Taxable → tax lines → Total to be paid,
                           with amount-in-words (Indian numbering);
                           internal allocations NEVER appear
  Schedule C             → dated payment rows from the SAME
                           computeContractEvents math the wizard
                           persists (cadence payments + seller-set
                           final payment included)
  Terms                  → the contract's TEXT BLOCKS — heading +
                           their real content (rich text preserved) —
                           plus one generated acceptance line
  Signatures / footer    → provider + buyer names; CNAK when known

ALSO WIRED
  ReviewSendStep now receives discountType/discountValue from wizard
  state, so the exported document shows the discount chain (the
  on-screen review step totals gain it via the same data on a later
  pass — the DOCUMENT is authoritative for the buyer).
