OWNER DECISION IMPLEMENTED:
  "We have a wizard pattern for contract creation — have an identical pattern
  for templates." A template IS a contract minus buyer and dates, so the SAME
  ContractWizard now runs in mode="template" with those steps removed. The
  old template builder page is NOT used any more by templates-list (stale-code
  candidate, untouched in this batch).

SUBMODULES AFFECTED:
  - contractnest-ui ONLY (3 modified files, no API/edge/DB changes)

FILES:
  1. contractnest-ui/src/components/contracts/ContractWizard/index.tsx
     - New props: mode="template", editTemplate, onTemplateSaved.
     - TEMPLATE_STEPS (all existing step components, same order as contracts):
         Contract Type → Acceptance → Details → Billing Cycle → Add Blocks
         → Billing View → Evidence Policy → Review & Save
       Skipped: path, counterparty (buyer), buyer assets, events preview.
     - Final action = "Save Template" → t_cat_templates via existing
       useSaveTemplate mutation. New templates save as DRAFT; sign-off stays
       on the templates list. NO contract records are ever created in
       template mode (contract draft auto-save is suppressed).
     - Storage: blocks[] in the standard {block_id, order, config_overrides}
       shape (for the VaNi match tier + wizard instantiation later);
       settings.wizard_state = full serialized wizard state (buyer/asset
       fields stripped) so EDITING A TEMPLATE ROUND-TRIPS EXACTLY;
       settings.defaults = nomenclature/duration/acceptance/billing/tax/
       evidence defaults for external consumers.
     - Edit hydration: opening a template restores its saved wizard state.
       Templates saved by other tools (no wizard_state) open with
       name/description/currency prefilled; blocks are re-picked.
     - Close mid-flow: same "Save as Draft?" dialog — in template mode it
       saves a draft TEMPLATE (wording adjusted), never a contract draft.
     - NOTE: editing a signed-off template keeps its signed-off status
       (flag if you'd rather it revert to draft on edit).
  2. contractnest-ui/.../ContractWizard/steps/ContractDetailsStep.tsx
     - Optional templateMode prop: hides the contract Status card (lifecycle
       lives on the templates list) and relabels "Contract Title" →
       "Template Title" with a template placeholder. Contract mode unchanged.
  3. contractnest-ui/src/pages/catalog-studio/templates-list.tsx
     - "Create New" and card click / Edit now open the ContractWizard in
       template mode (modal on the same page) instead of navigating to the
       old builder. List refreshes on save.
