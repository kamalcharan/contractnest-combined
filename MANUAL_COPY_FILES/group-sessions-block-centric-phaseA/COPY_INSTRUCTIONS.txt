WHAT CHANGED (the model)
----------------------------------------------------------------
The dashboard is now keyed by the GROUP-SESSION BLOCK, not by a flagged
"owner" contract.

  A "group session" = a catalog block with config->>'audience'='group'
  (m_cat_blocks). Its roster = the DISTINCT buyers of ACTIVE contracts
  (status='active') that carry that block (t_contract_blocks.source_block_id),
  scoped to the current environment (is_live).

Why: the reliable key is source_block_id — contract-level category/audience
markers drift (the same block shows category 'session' on one contract and
'service' on another). Keying on the block fixes the roster.

FIXES a real bug: the old roster was tenant-wide (counted everyone with any
membership contract). Live-verified example — "Bi Weekly Meetings":
  OLD dashboard: 3 members (Charan, Nikhil, Vasanth) — WRONG (Charan has no
                 group block)
  NEW dashboard: 2 members (Nikhil, Vasanth) — the contracts that actually
                 carry block 3f22e995
Card title is now the BLOCK name ("Bi Weekly Meetings"), and each roster row
shows the member's CONTRACT name + start/end dates.

NOTE: Occurrences / attendance % / QR are intentionally 0/empty here — the
shared per-block SCHEDULE arrives in Phase B (that's what fills Occurrences,
attendance and the holiday single-event edit). Roster + dues are live now.
