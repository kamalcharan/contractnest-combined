═══════════════════════════════════════════════════
Submodule affected: contractnest-ui  (UI only — no API / edge / DB changes)

WHAT THIS FIXES
─────────────────────────────────
1) Contract view → Equipment tab now HIDDEN for contracts that have no
   equipment/facility. Rule (chosen): show the tab only when the contract
   TYPE (nomenclature) is equipment/entity-based OR the contract already has
   equipment/facility rows. Pure-service contracts (e.g. "year contract",
   CN-1035) no longer show the Equipment tab or its "Add Equipment" button.

2) Contract view → Document tab (reuses ReviewSendStep):
   - Title was "Untitled Contract"  → now shows the real contract name
     (falls back to contract.name, which is where the title actually lives).
   - "Ref: #CN-XXXX" (hardcoded placeholder) → now shows the real number,
     e.g. "Ref: #CN-1035".
   - "Created <today>" (hardcoded to now) → now shows the real created date.
   - Timeline start date now uses the contract's real start_date.
   - Nomenclature chip now wired (shows only if the contract has one).
   NOTE: the creation Wizard's Review step is UNCHANGED in behaviour — it
   omits these new props, so it keeps the "#CN-XXXX" placeholder and "today"
   until the contract is actually created. All new props are optional.
