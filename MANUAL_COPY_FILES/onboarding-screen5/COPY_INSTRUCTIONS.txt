WHAT THIS SCREEN DOES
─────────────────────────────────────────────────────
Full VaNi-styled industry selection screen replacing the
placeholder IndustrySelectionStep from onboarding-screen4.

Features:
• Step 5 of 9 eyebrow
• VaNi intro bubble
• Search bar (filters visible industry list)
• 4-column grid of parent industries
  - Parent WITH children: shows "X specializations →" hint,
    clicking drills in to show child sub-segments
  - Parent WITHOUT children: directly toggleable (check mark)
  - Selected-child count badge shown on drillable parent cards
• Breadcrumb / Back to all industries when drilling
• Multi-select: click to toggle individual industries
• VaNi reaction bubble slides up after first selection
  (updates message based on count: 1 industry vs N industries)
• Pre-populates from already-saved served industries
• Floating action island:
  - Shows "X industries selected" / "Select industries you serve"
  - ← Back → /onboarding/theme-selection
  - Continue → addIndustries(selectedIds) → /onboarding/vani-consent
• Continue disabled before any selection
• Loading spinner while catalog / served industries load

DATA HOOKS
─────────────────────────────────────────────────────
useIndustries()           — industry catalog (from useProductMasterdata)
useServedIndustriesManager()  — pre-populate + addIndustries() call

SAVE PATTERN
─────────────────────────────────────────────────────
await addIndustries(Array.from(selectedIds))
→ calls POST /api/tenant-profile/served-industries with { industry_ids: [...] }
→ already production-ready with optimistic updates + rollback
