WHAT THIS BATCH DOES
────────────────────
This is the roster/member-view work promised after the Catalog Studio
attendance-policy card. Both already applied live and verified against
real BBB data (10 members, all currently over the 6-no-show cap in the
Test data — expected, since Test attendance wasn't tracked rigorously
before now):

1. DB (contractnest-edge) — migration 035:
   - gs_dash_roster and gs_member_block now compute, PER MEMBER:
     overall (occurrences in THEIR OWN contract window — join date
     through today/end date, excluding cancelled/skipped — not the
     block's total), attended, missed (= overall - attended),
     substituted (present rows flagged is_substitute), and the cap
     comparison (max_no_shows/max_substitutes + over_no_show_cap/
     over_substitute_cap booleans).
   - The cap is read from the MEMBER'S OWN contract snapshot
     (t_contract_blocks.custom_fields.config.groupSession.attendancePolicy)
     — not the live catalog block — so it reflects what that member
     actually signed, per the T&C design from the last batch.
   - migration 036: one-off backfill — the 10 existing Saturday Cadence
     member contracts predate the attendance-policy feature, so they
     had no policy in their own snapshot. Backfilled to match the
     block's real live policy (6/6) so there's real data to test
     against. Bi Weekly Meetings has no real member contracts, so
     nothing to backfill there.

2. UI (contractnest-ui):
   - useGroupSessionsDashboard.ts: GsRosterRow and GsMemberBlock types
     extended with overall/missed/substituted/max_no_shows/
     max_substitutes/over_no_show_cap/over_substitute_cap;
     GsMemberBlockAtt gains is_substitute.
   - Roster card (Group view): each row now shows {attended}/{overall}
     (member's own denominator, was wrongly the block's total before)
     plus a red "⚠ N/M no-shows" or "⚠ N/M subs" pill — only when that
     member is actually over a cap, so most rows stay clean.
   - Member view: KPI strip grows from 4 to 6 tiles (Attendance %,
     Attended, Last seen, Missed, Substituted, Dues), plus a new
     "Attendance policy" card comparing this member's actual counts
     against their own signed cap (green under cap, red at/over — only
     shown if a policy exists on their contract). The attendance-history
     grid gets a 4th marker: a purple "S" square for a session attended
     via substitute (was indistinguishable from a normal present ✓
     before).
