ROOT CAUSE (proven live, not inferred)
  TWO overloads of get_contract_by_id coexisted in the DB:
    • (uuid, uuid)                              — migration 060 body
    • (uuid, uuid, varchar, varchar)            — 060 body + CNAK access-key params
  The CNAK/global-access work grew the function's SIGNATURE; CREATE OR REPLACE
  only replaces a same-signature function, so Postgres created a SECOND one and
  nothing dropped the first. From that day, the edge's 2-named-arg call matched
  BOTH candidates → Postgres 42725 "function is not unique" (reproduced verbatim
  in SQL) → edge 500 RPC_ERROR → API default-mapped unknown codes to 400 → UI
  "Failed to load contract / Request failed with status code 400", and React
  Query retry:3 = the "auto refresh".
  Scope: detail view broken for EVERYONE (seller AND buyer — same endpoint,
  role is client-side). The contracts LIST was never broken (different query).
  "Nothing changed recently" was true — the duplicate dates to the CNAK-era SQL;
  it went unnoticed because later sessions never opened /contracts/:id.
