FIX: CNAK not showing on contract success screen (draft flow)
═══════════════════════════════════════════════════

PROBLEM:
When a contract is created from a draft (draft → active or draft → pending_acceptance),
the CNAK is generated in the DB but NOT returned to the frontend. The success screen
shows no CNAK info because contractResult.global_access_id is still null.

ROOT CAUSE:
update_contract_status RPC generates CNAK during draft transitions but does not
include global_access_id in its response JSON. The frontend never picks it up.

FIX (2 parts):
1. SQL: Add global_access_id to update_contract_status response
2. UI: Read global_access_id from updateStatus response and set it on contractResult
