WHAT CHANGED — Screen8APricingStep.tsx  (FULL REPLACEMENT):
──────────────────────────────────────────────────────────────
Was: Hardcoded mock data (Hydraulic Lift, MRL Traction Lift — Lifts & Elevators)
Now: Real API-driven implementation

- Fetches blocks: GET /api/catalog-studio/blocks?tags[]=industryId1&...
- Filters client-side to is_seed=true blocks
- Groups blocks by resource_template_id
- Template display name resolved from selectedEquipmentTemplates (passed via routeState)
  or parsed from block name ("Basic AMC — Hydraulic Lift" → "Hydraulic Lift")
- Tier detected from block name: "premium"/"cmc" → tier 2, "comp" → tier 1, else tier 0
- Anchor step: user sets price for first block (basic tier of first equipment group)
- Rate card: all blocks shown with computed prices (anchor × tier multiplier: 1×, 1.5×, 2.3×)
- Any price is editable inline
- On confirm: PATCH /api/catalog-studio/blocks/:id with { config: { base_price, currency_id } }
- Empty state if no blocks found (with graceful "Continue anyway" button)
- Loading spinner while fetching
- Save error message on PATCH failure
- Saving spinner in action island button
