WHAT THIS DOES
─────────────────────
Persists audience:'group' on contract events so the ops calendars can tell a
group session from a 1:1 visit. Then:
  - Event Schedule rows show "Group Session" (Users icon) and a "Check-in" button
    instead of "Book appointment".
  - A new "Group sessions" tab filters the list to audience='group'
    (same mechanism as the Service / Billing tabs).

audience flows end to end:
  UI computed_events (payload) -> create_contract_transaction (stores verbatim)
  -> process_contract_events_from_computed (passes verbatim)
  -> insert_contract_events_batch (writes audience) -> t_contract_events.audience
  -> get_contract_events_list (returns audience) -> the Event Schedule row.
