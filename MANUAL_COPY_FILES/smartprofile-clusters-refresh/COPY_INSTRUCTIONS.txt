═══════════════════════════════════════════════════
Branch (combined repo): claude/handover-templates-agent-qretm1
Submodule affected: contractnest-ui  (only)

ROOT CAUSE
  The view page reads its cluster list from the PROFILE query cache
  (smartProfileData.clusters). But "Save Clusters" only invalidated the
  separate CLUSTERS query — never the profile query. So the profile cache
  kept the stale empty cluster list from before you generated them, and
  navigating back to the view showed 0 clusters (even though they were saved
  in the DB and returned correctly by the edge GET).

  Confirmed: hub@h.com (1f0a8dd2) has 5 clusters saved in t_semantic_clusters,
  and the deployed edge GET returns them nested under smartprofile.clusters.
  Purely a client cache-refresh gap.

FIX (handleSaveClusters)
  After saving clusters, also invalidate the PROFILE query
  (smartProfileQueryKeys.profile(tenantId)) so it refetches with the saved
  clusters. Now the view shows them immediately and after navigating away/back.
