REQUEST: on a contract's Tasks tab, a Group Session task should never
show "Book appointment" or "Start Service" — those only make sense for
individually-delivered services. Instead it should show a heading and a
link to navigate to the Group Session menu.

WHAT I FOUND:
  - EventCard.tsx already correctly excluded "Book appointment" for
    group-session events (`canBookAppointment` already required
    `!isGroupSession`) — but left that entire row blank for group
    sessions instead of showing anything useful there.
  - SellerTasksTab.tsx's separate "Start Service" bulk-action button (one
    per date, opens a panel to start every service event scheduled that
    day) did NOT exclude group-session events at all — it would have
    included them in that bulk action if any existed in that date group.
  - Checked CN-1021 (the contract in your screenshot) directly: its
    "Saturday Cadence" group-session block has ZERO rows in
    t_contract_events — group sessions generate their occurrences into a
    separate table (t_group_session_schedule) surfaced on the dedicated
    Group Session dashboard instead, by design. So this exact bug
    couldn't be reproduced live on THIS contract right now (there's no
    group-session task card to show at all) — the fix is for whenever a
    group-session event DOES appear in this list (some other flow may
    still route one through here, and it's the correct general fix
    regardless).

FIX:
  - EventCard.tsx: added a new block, shown only when
    event.event_type === 'service' && isGroupSession — a "Group Session"
    label plus a "Go to Group Session" button that navigates to
    /group-sessions (the existing Group Session dashboard route). The
    existing "Book appointment" block now also explicitly excludes
    isGroupSession (was already effectively excluded via
    canBookAppointment, made explicit for clarity).
  - SellerTasksTab.tsx: hasIncompleteService and the event list passed to
    handleStartService both now filter out events with audience==='group'.
