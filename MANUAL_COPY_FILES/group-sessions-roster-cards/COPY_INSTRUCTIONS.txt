WHAT CHANGED (per your screenshot + "every member is a card, it will
have attendance and small statistics like 6/8, BBB will have average
60 people in each chapter")
────────────────────
The previous version of the Roster page showed a numeric table-style
list (Attended/Missed/Subbed as separate columns). That wasn't what
you asked for — you wanted every member shown as a CARD with a compact
"6/8" style stat AND their actual attendance-history grid (✓/✕/S
squares), all stacked and visible by default, searchable by name.

1. DB — migration 037: gs_dash_roster now also returns each member's
   full occurrence-by-occurrence attendance array (same shape
   gs_member_block already computes per-member), so ~60 members' grids
   load in ONE call instead of 60 separate API calls. Applied live,
   verified against real BBB data (26-occurrence grid returned
   correctly for MALLADI MADHU KUMAR).

2. UI — Roster page member list is now cards, not a table:
   - Each card: avatar, name, contract name, a compact "attended/overall"
     stat (e.g. "0/8", red if over cap), a cap-status pill (only shown
     as a warning when actually over cap — from that member's own
     signed policy, nothing hardcoded), a dues pill, "View contract".
   - Below that: the member's own attendance-history grid — the same
     ✓ (present) / ✕ (absent) / S (attended via substitute) / ·
     (upcoming) squares used everywhere else in this feature, just
     smaller (to stay compact across ~60 cards).
   - All cards shown by default (not click-to-expand) — the search box
     narrows which cards are visible as you type.
