WHAT THIS IS
----------------------------------------------------------------
Adds an "Offline UPI" provider under the Payment Gateway integration type.
It is NOT a real gateway (no API keys, no fees) — it just stores the tenant's
UPI VPA + payee name so the group-session check-in can later open the member's
UPI app (GPay/PhonePe) via a upi:// intent (that's G4).

Because /settings/integrations renders providers dynamically from
config_schema.fields and saves values into t_tenant_integrations.credentials,
seeding this ONE provider row is all G1 needs — no UI or API changes.

Config fields the tenant fills (Settings → Integrations → Offline UPI):
  - UPI ID (VPA)   e.g. bbbchapter@okhdfcbank
  - Payee name     shown in the member's UPI app
metadata.kind='upi_intent' flags it so the check-in "Pay now" (G4) knows to
build a UPI intent rather than call a gateway.
