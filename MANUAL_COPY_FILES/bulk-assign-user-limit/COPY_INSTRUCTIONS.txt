BUG REPORT: in /catalog-studio/templates-list, "bulk assign" only ever
showed the same 20 users, even after reopening/reloading.

ROOT CAUSE: BulkAssignDialog.tsx called useContactList({ ..., per_page:
100, ... } as any). useContactList's ContactFilters type has no
`per_page` field — only `limit`. The `as any` cast let this typo compile
silently, so `per_page: 100` was just dropped and no limit was ever sent.
With no limit param, three separate layers each fell back to their own
default of 20 (contractnest-ui/src/hooks/useContacts.ts:29,
contractnest-api/src/controllers/contactController.ts:54/66,
contractnest-edge/supabase/functions/_shared/contacts/contactService.ts:46)
— consistently, which is why reopening the dialog kept returning the same
20 (same default params -> same first page every time).

FIX: renamed `per_page: 100` to `limit: 100` (the field the hook actually
reads) and dropped the `as any` cast now that the object is fully valid
against ContactFilters — leaving that cast in place would have hidden the
exact same class of typo again in the future.

NOTE: the API/edge layers still cap `limit` at 100 (Math.min(limit, 100))
regardless of what's requested, and BulkAssignDialog has no pagination/
"load more" UI — it just renders whatever comes back. That's fine for a
tenant with under 100 assignable contacts; if a tenant ever has more, a
follow-up would be needed (page size selector or infinite scroll) to see
everyone. Not addressed here — this batch only fixes the specific
"same 20 users" bug reported.
