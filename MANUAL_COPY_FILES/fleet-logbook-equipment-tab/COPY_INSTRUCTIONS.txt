WHAT CHANGED
────────────
The contract detail Equipment tab is redesigned as a "Fleet Logbook"
(design reference: SPRINT_REFERENCES/8-fleet-logbook-contract-view.html).
All data is joined CLIENT-SIDE from existing queries — no backend change.

1. Fleet summary strip — machines covered (equipment/facility breakdown,
   placeholders counted separately), awaiting-asset count (doubles as the
   "show only awaiting" filter toggle), fleet-wide visits proven X/Y,
   next visit due date, machines behind schedule.
2. Category-grouped machine grid — cards grouped under category headers
   ("Air Conditioners · 6 units · 5 attached · 1 awaiting · 14/24 visits
   proven"). Each attached card gains a compact service-state footer:
   "X of Y visits proven · Next: <date>" + thin progress bar (red when
   any visit is overdue) + "Last proven <date> · <assignee>". Placeholder
   cards show "N visits locked until an asset is attached" and keep the
   existing pill + Attach Asset button.
3. Lens toggle — segmented control "Machine cards | Service matrix".
   Matrix: rows = machines (grouped, name + serial), columns = the
   contract's distinct service visit dates (falls back to "Visit N"
   labels beyond 14 dates; horizontal scroll), cells = status dots
   (proven green / due amber / overdue red / scheduled grey / locked
   striped) with a legend. Read-only. Toggle hidden when the contract
   has no service visits.
4. Machine logbook drawer — clicking a machine card (not its buttons)
   opens a right-side slide-in drawer listing that machine's visits
   chronologically: block name + visit n of m, date, visit status,
   per-asset row status chip, assignee, proven date. Read-only; closes
   on overlay click or X.
5. Stay-open attach — in attach-placeholder mode, a successful attach
   no longer auto-closes the picker while other placeholders remain: it
   advances to the next placeholder (same category preferred) and shows
   an "Attached '<asset>' — now attaching: <category>" note. Closes only
   when no placeholders remain or the user closes it.
6. "Show only awaiting" filter — chip next to the lens toggle AND the
   strip's Awaiting stat both toggle a placeholder-only view of the grid.

Data join (EquipmentTab):
  useContractEventsForContract(contractId, per_page 200) +
  useContractEventAssets(contractId). A row belongs to a machine when
  row.asset_ref === detail.id OR row.asset_ref === detail.asset_registry_id.
  Rows join to events by event_id for scheduled_date/status. Per machine:
  provenCount, totalVisits, overdueCount (row not proven and event
  overdue/past-date and not completed/cancelled/skipped), nextDueDate
  (earliest non-proven date >= today), lastProven (date + assignee).

FIX (19 Jul, post first user test): per-asset rows are NO LONGER required
for the fleet view to render. Contracts whose service events have no
t_contract_event_assets rows (anything created before Sprint 3 seeding,
e.g. CN-1020) previously fell back to the plain old card grid — strip
stats showed "—", no footers, no matrix, no drawer. Now every service
event with no per-asset row for a machine synthesizes a visit for that
machine from the event alone (proven = event completed, overdue from
event status/date). Per-asset rows, when present, refine the same visit
to true per-machine grain. Matrix columns + hasServiceData now derive
from service events directly. A contract with equipment but genuinely
zero service events shows an explicit dashed note instead of silently
hiding the service layer.
