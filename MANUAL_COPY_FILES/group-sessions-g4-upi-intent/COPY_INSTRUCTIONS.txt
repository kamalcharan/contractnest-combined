WHAT THIS DOES
----------------------------------------------------------------
When a member selects an open due, the check-in page now shows:
  - a "Pay ₹X now" button -> opens the phone's UPI app (GPay/PhonePe/Paytm)
    via a upi:// intent, pre-filled with the chapter's VPA, payee, amount and
    a note (chapter — due label);
  - a "scan to pay" QR (same upi:// string) + the VPA text, as a desktop
    fallback (no UPI app on a laptop);
  - the existing UPI-reference field. The declaration flow is UNCHANGED — the
    chair still confirms offline. This is a convenience layer, not a new rail.
The Pay UI only appears if the tenant has configured Offline UPI (G1).

INPUT-FOCUS HOTFIX (also here)
----------------------------------------------------------------
Shell/Card wrappers were defined inside the page component -> React remounted
the tree on every keystroke (inputs lost focus). They are now module-scope.
If you already applied the standalone hotfix, this file supersedes it.

HOW THE VPA REACHES THE PUBLIC PAGE
----------------------------------------------------------------
Integration credentials are AES-GCM encrypted and never returned to clients.
A VPA + payee are payer-facing (non-secret), so for config_only providers we
keep a plaintext copy at credentials->'public'. New RPC
gs_checkin_payment_config(token) returns only that slice for the token's
tenant. API: GET /api/checkin/:token/payment-config.
