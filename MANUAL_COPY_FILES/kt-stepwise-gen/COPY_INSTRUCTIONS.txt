COPY INSTRUCTIONS — kt-stepwise-gen
Feature: Stepwise KT generation — 3 independent on-demand steps
=====================================================

NOTE: This supersedes kt-reduce-tokens for overlapping files.
Apply kt-stepwise-gen AFTER kt-reduce-tokens, or apply only kt-stepwise-gen
(it includes all kt-reduce-tokens changes for API files).

SUBMODULES AFFECTED:
  - contractnest-api
  - contractnest-ui

HOW IT WORKS:
  Step 1 — + Variants    → POST /api/knowledge-tree/generate-variants
  Step 2 — + Parts       → POST /api/knowledge-tree/generate-spare-parts  (requires variants)
  Step 3 — + Checkpoints → POST /api/knowledge-tree/generate-checkpoints  (requires variants)
  + Overlays             → POST /api/knowledge-tree/generate-overlays      (unchanged)
  + Install / + Decomm   → POST /api/knowledge-tree/generate              (unchanged, uses existingKT:true)

FAILURE UX:
  - If Step 2 fails: toast shows "Step 1 (Variants) ✓ (N) | Step 2 (Spare Parts) failed: [msg] — click '+ Parts' to retry"
  - If Step 3 fails: toast shows "Step 1 ✓ | Step 2 ✓ | Step 3 (Checkpoints) failed: [msg] — click '+ Checkpoints' to retry"
  - No automatic retry. Each step button stays active for manual retry.

TOKEN BUDGET PER STEP:
  Step 1 (variants):           max 2,000 tokens  → near-zero failure risk
  Step 2 (spare parts + map):  max 6,000 tokens  → very reliable
  Step 3 (checkpoints+cycles): max 10,000 tokens → reliable
  vs previous single prompt:   16,000 tokens     → was hitting limit
