WHAT CHANGED
  #4 Template mode = illustrative/read-only (honesty).
     Previously you could edit event dates in TEMPLATE mode; they highlighted and
     counted "N adjusted" — then were SILENTLY discarded on save. Now, in template
     mode the Events Schedule shows dates as static chips (no editing, no reset),
     and the footer reads "Illustrative schedule — this preview assumes a start
     today; real dates are set on each contract created from this template."
     Wired via a new EventsPreviewStep `readOnly` prop; VaNiReviewFinalize passes
     readOnly={isTemplateMode}. (Contract mode is unchanged — fully editable and
     the overrides persist as before.)

  #5 Persisted event order now matches the preview.
     computeEventsForApi applied date overrides but never re-sorted, so a moved
     event kept its old slot in computed_events. Now it re-sorts by scheduled_date
     after overrides — persisted order == the (re-sorted) preview.

  #6 EMI installments reconcile to the exact total.
     The final EMI installment now absorbs rounding, so the installments sum
     EXACTLY to grandTotal (no ±cents drift).

  #7 Robust event ids.
     makeEventId used blockId.slice(0,8) — two blocks sharing an 8-char id prefix
     could collide, misapplying date overrides / clashing React keys. Now uses the
     full blockId. (Event ids are not persisted in computed_events, so no migration.)

  + Dropped the misleading "Service & Parts" column label -> "Service" (spare_part
    events are never emitted by the derivation).

  • Per-block recurring rounding reconciliation (same idea as EMI #6, different
