LATEST FIX 2 (post-review, "how does user know what's selected at 100
blocks"): added a green "✓ Selected (N)" chip to the filter row, mutually
exclusive with the type chips — clicking it shows every selected block
across ALL types, no pagination cap. The sticky bottom bar's block count
is now also clickable and jumps straight into this same view. Selecting
a type chip (or "All") exits Selected view.

LATEST FIX (post-review): the "Filter:" row was replaced. The earlier
version had two abstract filters ("For <coverage type>" resource-scope
match, "Recommended only") that were confusing and, on a thin/test
catalog, could make the list look empty. Replaced with a plain TYPE
filter — one chip per block type with a live count: All / Services (N) /
Spare Parts (N) / Fees & Billing (N) / Terms & Checklists (N) / Media &
Documents (N). Click a chip to show only that type (multi-select — click
more than one to combine); click "All" to clear. This is the intended
lever at real catalog volume: pick a type, then use the search bar to
find the exact block inside it. No "recommended" or "relevance" concept
left in the filter row — VaNi's suggestions still float to the top of
each section and still have their own inline banner, unrelated to this
filter.

WHAT CHANGED
────────────
The wizard's "Add Service Blocks" step is rebuilt to the approved mock 9
anatomy. ONLY THE RENDER CHANGED — every piece of selection/pricing
logic is the same code that powered the old 3-column layout:
buildConfigurableBlock (pricing records, tax, cadence, group sessions),
handleAddBlock / handleAddBlocks / handleAddFlyByBlock /
handleRemoveBlock / handleUpdateBlock, the two duration-sync effects
(group-session auto-count, cadence money sync + toast), the unified-
cycle mismatch detection, the duplicate guard, all toasts, RFQ mode.

Mock elements delivered:
1. Single centered column (max 780px), "tick what this contract
   delivers" header.
2. "Blocks attach to: <Type> (n) · <Type> (n)" context line — the bold
   names ARE the coverage scope switcher (same per-coverage instance
   model as before: blockId__coverageTypeId).
3. VaNi inline banner — "VaNi suggests N blocks … [Add all N] [I'll pick
   myself]". Same deterministic heuristic as the recommender drawer,
   extracted to serviceBlocksChecklist/recommend.ts (single source).
4. Full-catalog search bar (name / description / type).
5. Catalog as a checklist, grouped: Services / Spares & parts / Fees &
   billing / Terms & checklists (no price — "shapes the document") /
   Media & documents (video, image, document → COMING SOON, disabled
   per MVP scope). "x of N shown · show all N" per section; selected
   rows always visible; VaNi-recommended rows float up.
6. Tick = add (same toast), untick = remove. Checked rows show the
   compact summary line: visits tag, cycle tag (amber when it offends
   unified billing), green discount tag, Billing-only tag, Edit.
7. Edit opens the 3-field editor: Visits (+Unlimited), Your price (live
   "List ₹X → −Y% recorded as discount" hint), Billing cycle (cadence
   rate-card options for cadence-priced blocks — switching restores
   per-cadence overrides exactly like the old card; PrePaid/PostPaid/
   Custom otherwise). Advanced disclosure: tax (read-only), service
   interval (read-only), show-description toggle, billing-only toggle,
   custom cycle days. Done closes.
8. ONE amber mismatch banner with one-click "Align all to <majority>" —
   snaps every offender through the same price math (cadence blocks
   without that rate are skipped with a toast).
9. "+ Add a custom line (not in your catalog)" dashed button → FlyBy
   service line, rendered as an editable checklist row (name,
   description, price, cycle, remove). RFQ mode = custom lines only,
   no pricing shown.
10. Sticky bottom total bar: "N blocks · ₹X term total incl. tax ·
    discount recorded on N blocks" + Continue. Continue calls the
    wizard's OWN validated handleNext (passed down as onContinue) — no
    duplicate navigation logic. The wizard's regular action bar is
    unchanged and still works.

Superseded by the mock (removed from this step, files untouched):
  - 3-column layout (BlockLibraryMini panel, ContractPreviewPanel
    column) — preview still exists on Review & Send.
  - Drag-to-reorder of added blocks (mock has no drag; block order =
    selection order, as before for new adds).
  - VaNi drawer trigger button — the inline banner covers it; the
    drawer component file is untouched.
