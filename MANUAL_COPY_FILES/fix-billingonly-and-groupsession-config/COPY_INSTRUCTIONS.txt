#2  BILLING-ONLY BLOCKS SHOULD NOT CREATE SERVICE EVENTS  (contractnest-api)
────────────────────────────────────────────────────
Root cause: the SERVER event-derivation path (used by assign/composer) lacked
the billingOnly guard the UI has, AND the composer stripped the flag and
hardcoded categoryId:'service' / unlimited:false.

Fix (no hardcodings — the real flag is carried end to end):
  - contractEventsDerivationService.ts
      * DerivationBlock: added `billingOnly?: boolean`
      * SERVICE EVENTS loop: added `if (block.billingOnly) continue;`
  - contractComposerService.ts
      * CandidatePayload: added `billing_only?: boolean`
      * shortlist candidates: billing_only = block.config?.billingOnly === true
      * template candidates:  billing_only = b.config?.billingOnly === true
      * PrefillBlock.config: added `billingOnly?`
      * prefill mapping: config.billingOnly = cand.billing_only
      * derivation mapping: now passes billingOnly + real categoryId/unlimited
        (removed the hardcoded 'service' / false)
  - __tests__/contractEventsDerivationParity.ts
      * RefBlock + reference impl guard + a new "membership dues" fixture/scenario
        so the parity contract now ENFORCES billing-only = zero service events.

────────────────────────────────────────────────────
#3  GROUP-SESSION CADENCE (14 days / Saturday) NOT COMING INTO THE BLOCK  (contractnest-ui)
────────────────────────────────────────────────────
Root cause: catalog block "Bi Weekly Meetings" is stored with category='session'
and config={"icon":"Wrench"} — the cadence/anchor/audience were WIPED. Why:
the create path remaps session->'service' (so buildServiceConfig runs), but the
UPDATE path did NOT — it saved category='session' and called buildConfig('session'),
which has no case and returns just { icon }. So EDITING a group-session block
wiped its whole config.

Fix:
  - utils/catalog-studio/catBlockAdapter.ts  (blockToUpdateData)
      * added the same session->'service' remap used by blockToCreateData, applied
        to data.type/data.category AND the buildConfig() call.
