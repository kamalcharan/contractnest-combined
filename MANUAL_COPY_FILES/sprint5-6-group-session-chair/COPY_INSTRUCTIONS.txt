CORRECTION, READ THIS FIRST
────────────────────────────────────────────────────
The previous batch (sprint5-6-group-session-occurrence) built a brand-new
table (t_session_occurrence_assignments) + 2 RPCs to solve "assign one
instructor per Group Session occurrence, fan out to every member" —
reasoning from t_contract_events/t_appointments alone, WITHOUT first
searching the codebase for an existing Group Sessions backend. The owner
then showed a live Group Sessions dashboard (localhost:5173/group-
sessions, Test environment) that the earlier live-only DB queries had
completely missed.

Ground truth: Group Sessions already have a full, mature backend
(bbb-foundation/021 onward, ~26 gs_* RPCs) — occurrences already live in
t_group_session_schedule (ONE row per tenant+block+date, generated by
gs_generate_schedule() from the catalog block's cadence config,
decoupled from t_contract_events entirely — member contracts carry the
block via t_contract_blocks, they do NOT spawn one event per member).
Attendance, roster, QR check-in (self/guest/substitute), dues-paid
marking, and a dashboard all already exist and are live.

The earlier batch was REDUNDANT with this — a parallel occurrence
concept solving an already-solved problem. It has been REVERTED live
(0 rows ever existed, 0 dependents, confirmed before dropping) and its
MANUAL_COPY_FILES folder deleted. This batch replaces it.

WHAT THIS BATCH ACTUALLY DOES
────────────────────────────────────────────────────
The ONE real gap in the existing system: t_group_session_schedule had no
column for who chairs/hosts a given occurrence — exactly what the owner
asked for ("a Group Session is headed by someone... he will have an
appointment against 'Group Session' in the cadence").

- ALTER t_group_session_schedule: adds assigned_to (uuid),
  assigned_to_name (text). No new table.
- gs_dash_occurrences() (existing RPC): additive change, now also
  returns assigned_to/assigned_to_name per occurrence. Everything else
  reproduced verbatim from the live definition.
- New RPC gs_schedule_assign(tenant, occurrence_id, assigned_to,
  assigned_to_name) — sets who chairs ONE occurrence (referenced by its
  own schedule-row id, since occurrences already have one), returns the
  refreshed occurrence list via gs_dash_occurrences(). Matches the
  existing gs_schedule_status/gs_schedule_move convention exactly (same
  {ok:false, reason:'not_found'} error shape, no exception wrapper —
  this RPC family doesn't use one).

This is a ~15-line fix, not a new subsystem — because the subsystem
already existed.
