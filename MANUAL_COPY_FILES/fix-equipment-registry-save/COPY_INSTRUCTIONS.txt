WHAT WAS BROKEN:
═══════════════════════════════════════════════════

Bug 1: API validator rejected null values for condition/criticality
  - express-validator .optional() only skips undefined, NOT null
  - When assets had no condition or criticality, they sent null
  - Validation failed → 400 error → contract creation aborted

Bug 2: create_contract_transaction RPC missing 3 columns
  - start_date was never inserted (fell back to DB default)
  - allow_buyer_to_add_equipment was never inserted (always false)
  - coverage_types had no DB column at all (silently lost)

Bug 3: update_contract_transaction RPC also missing the same 3 columns
  - Editing a contract would lose start_date, equipment flag, and coverage data

Bug 4: get_contract_by_id RPC didn't return coverage_types
  - Even if saved, coverage data would not appear when loading contract details
