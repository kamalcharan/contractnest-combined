WHAT WAS BROKEN:
═══════════════════════════════════════════════════

Bug 1: API validator rejected null values for condition/criticality in contractValidators
  - express-validator .optional() only skips undefined, NOT null
  - When assets had no condition or criticality, they sent null
  - Validation failed → 400 error → contract creation aborted

Bug 2: create_contract_transaction RPC missing 3 columns
  - start_date was never inserted (fell back to DB default)
  - allow_buyer_to_add_equipment was never inserted (always false)
  - coverage_types had no DB column at all (silently lost)

Bug 3: update_contract_transaction RPC also missing the same 3 columns
  - Editing a contract would lose start_date, equipment flag, and coverage data

Bug 4: get_contract_by_id RPC didn't return coverage_types
  - Even if saved, coverage data would not appear when loading contract details

Bug 5: GET /api/client-asset-registry returning 400 from wizard step 5
  - Controller was passing undefined params (id, resource_type_id, status, offset)
    to the edge function, which could cause serialization issues
  - Also was always sending is_live=true as a boolean param
  - Fixed by only passing defined, truthy params to the service layer
  - Added diagnostic logging for future edge function errors

Bug 6: POST /api/client-asset-registry returning 400 from wizard step 5
  - clientAssetRegistryValidators.ts had the SAME .optional() bug
  - transformForAPI in clientAssetRegistryService converts empty/undefined to null
  - .optional() only skips undefined, so null values hit .isUUID()/.isISO8601() etc.
  - All nullable fields now use .optional({ values: 'null' })
