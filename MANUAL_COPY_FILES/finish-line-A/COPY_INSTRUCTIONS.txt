WHAT CHANGED (A items)
  #1 Unified template serializer (highest-value).
     VaNiReviewFinalize.handleSaveAsTemplate now builds wizard_state the SAME way
     the wizard does — createInitialWizardState() fills every field,
     sanitizeStateForTemplate() strips buyer/assets/overrides, serializeWizardState()
     normalises dates. Both serializers are now exported from ContractWizard/index.
     -> a VaNi-saved template hydrates CLEANLY when later opened in the wizard
        (identical wizard_state shape on both save paths).

  #2 Contract-level tax carries through the template fast path (API).
     assembleFromTemplate now applies the template's defaults.selected_tax_rate_ids
     to the subtotal (fetched via taxSettingsService.getTaxRates) — setting
     selectedTaxRateIds + taxBreakdown + taxTotal + grandTotal. SAFE GUARD: it only
     applies when per-block tax came out ZERO, so a template that already taxes per
     block is never double-taxed.

  #3 Timezone-correct event grouping.
     groupEventsByDate (and the timeline React key) now key on the LOCAL calendar
     day instead of UTC — an event near local midnight no longer lands under the
     wrong day / splits a day for non-UTC tenants.

  #4 Per-block recurring installments reconcile.
     Recurring per-block billing now makes the final period absorb rounding, so a
     block's recurring billing sums EXACTLY to its total (matches the EMI fix).
