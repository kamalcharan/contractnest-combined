BATCH: sprint2-step1-checkpoint-field-type
SUBMODULE(S) AFFECTED: contractnest-ui

WHAT THIS BATCH IS
-------------------
Step 1 of the Sprint 2/7 HVAC pilot POA. Adds a real "Checkpoint" field type
to the Smart Forms builder -- renders as three traffic-light buttons
(Good / Attention / Fail, green/amber/red) instead of a generic dropdown,
per the owner's confirmed decision on the checklist+service-form playground.
Optional "photo required" flag shown as a small camera indicator.

Backend needs no changes: contractnest-edge/supabase/functions/smart-forms/
index.ts's schema validator only checks structural shape (id/title/sections/
fields present, no duplicate ids) -- it has no allow-list of field.type
values, so 'checkpoint' passes validation with zero backend work.

  - Added `photo_required?: boolean` to the FormField interface.

contractnest-ui/src/pages/admin/smart-forms/utils/fieldTypeRegistry.ts
  - New 'checkpoint' entry in the Selection Fields category (🚦 icon),
