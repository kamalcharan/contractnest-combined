WHAT THIS ADDS (three parts)
----------------------------------------------------------------
D1. ENFORCEMENT GUARDS (your rules 1 & 3)
    - Rule 1: the ContractWizard block library HIDES group-session blocks on the
      DIRECT contract path (mode='contract'); they remain available only in
      TEMPLATE mode. (BlockLibraryMini hideGroupBlocks; threaded from
      ContractWizard/index -> ServiceBlocksStep.)
    - Rule 3: adding a group-session block to a template is BLOCKED if that block
      already belongs to another template (a group session = one template only).
      Uses useCatTemplates to compute the used set (excludes the template being
      edited); rejects with a warning toast in single + bulk add.

D2. QR IMAGE (self-contained, zero new dependency)
    - New src/utils/qrcodegen.ts — a vendored QR encoder (nayuki qrcodegen, MIT),
      NO npm install needed. Verified: v1->21, v6->41, finder/timing/separator ok.
    - New CheckinQrModal: dashboard "Open check-in" now opens a modal with the
      scannable QR + link (copy) + Download PNG / Download SVG / Open. Print the
      PNG as the chapter poster.

D3. CHAIR DUES CONFIRMATION (reuses existing endpoints)
    - Dashboard overview shows "Payments to confirm" — members' UPI payments
      declared at check-in — with Confirm (marks the billing event PAID) / Reject.
    - Reuses /api/session-checkin/declarations + /declarations/:id/confirm
      (already mounted by session-checkin-route-fix). No new API.

Production checklist:
  Transaction Mgmt : confirm flips declaration + billing event server-side (RPC)
  Race conditions  : mutations invalidate + refetch; buttons disable while pending
  Error handling   : mutation onError toasts; QR renders '—' fallback on failure
  Toasts/Loaders   : react-hot-toast on confirm/reject; disabled busy states
  Enforcement      : additive optional props (default off) — no behaviour change
                     outside the group-session paths

TYPECHECK: tsc --noEmit clean across all touched files (wizard incl.).
