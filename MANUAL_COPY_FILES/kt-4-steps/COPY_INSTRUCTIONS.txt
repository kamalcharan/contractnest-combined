📦 FEATURE: kt-4-steps — 4-Step KT Generation
═══════════════════════════════════════════════════

PROBLEM SOLVED:
  The old single-prompt KT generation hit max_tokens limits:
  - step3-checkpoints-pm: out=10000, stop=max_tokens → service_cycles missing → 500
  - step2-spare-parts:    out=6000,  stop=max_tokens → spare_part_variant_map:1 entry
  Each step now generates EXACTLY ONE array type, saving to DB immediately.
  If a step fails, prior steps are already in DB and can be retried independently.

ARCHITECTURE (4 separate API calls, each with its own token budget):
  Step 1 → /api/knowledge-tree/generate-variants      (max_tokens: 2000)
           Generates: variants[]
  Step 2 → /api/knowledge-tree/generate-spare-parts   (max_tokens: 10000)
           Reads:    real variant UUIDs from DB
           Generates: spare_parts[] + spare_part_variant_map[]
  Step 3 → /api/knowledge-tree/generate-checkpoints   (max_tokens: 8000)
           Reads:    real variant UUIDs from DB (for context)
           Generates: checkpoints[] + checkpoint_values[] ONLY (no service_cycles)
  Step 4 → /api/knowledge-tree/generate-service-cycles (max_tokens: 4000)
           Reads:    real checkpoint UUIDs from DB
           Generates: service_cycles[]

FILES CHANGED:
  contractnest-api:
    src/skills/kt-checkpoints-generator.md        (max 15 checkpoints, 3 values, NO service_cycles)
    src/skills/kt-service-cycles-generator.md     (NEW — service_cycles only)
    src/services/knowledgeTreeGeneratorService.ts (added generateServiceCycles method)
    src/controllers/knowledgeTreeController.ts    (added generateServiceCycles handler)
    src/routes/knowledgeTreeRoutes.ts             (added /generate-service-cycles route)

  contractnest-edge:
    supabase/functions/knowledge-tree/index.ts    (save_mode:'service_cycles' added)

  contractnest-ui:
    src/hooks/queries/useKnowledgeTree.ts         (useKTGenerateServiceCycles hook added)
    src/pages/service-contracts/templates/admin/knowledge-tree/KnowledgeTreeDetail.tsx
                                                  (4-step cascade + "+ Cycles" button)

Submodules Affected: contractnest-api, contractnest-edge, contractnest-ui

Production Checklist:
  ✅ Transaction Management: Each step saves via edge function with save_mode selective wipe
  ✅ Race Condition Handling: Each step is independent; DB is source of truth for UUIDs
  ✅ Error Handling: Try-catch in each step handler; cascade stops on failure
  ✅ Toasts: vaniToast.success/error on each step completion
  ✅ Loaders: Loader2 spinner on each step button while pending
