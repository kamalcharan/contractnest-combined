DESIGN DECISIONS
─────────────────────────────────────────────────────
- BusinessDetailsStep lives inside OnboardingLayout so the progress
  header chrome is visible (unlike VaNi Intro which is full-screen).
- submitProfile() from useTenantProfile hook is NOT used because it
  validates business_type_id + industry_id which are collected later.
  Instead, api.put/post is called directly with only the fields
  collected on this screen.
- GST/PAN are cosmetically important for Indian SMB users but no DB
  column exists yet. They're shown but their values aren't persisted.
  When the column is added, just include them in the API payload.
- City chips are a multi-select → joined as "Mumbai, Delhi" in `city`.
