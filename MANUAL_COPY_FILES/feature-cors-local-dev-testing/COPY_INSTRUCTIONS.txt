📦 FIX: Allow Expo web dev origin through API CORS
═══════════════════════════════════════════════════
Branch folder : feature-cors-local-dev-testing
Submodule     : contractnest-api (ONLY)
File changed  : src/index.ts — CORS allowedOrigins fallback array
Verified      : Additive-only change, 4 lines, no logic changed

WHY
  Testing the ContractNest mobile app via `npm run web` opens a real
  browser (localhost:8081), which enforces CORS. The API's fallback
  origin allowlist only had localhost:3000 / 5173 (the web app's Vite
  ports), so every fetch from the mobile app's web preview was silently
  blocked by the browser → showed as "Network error" in the app.
  Native testing (Expo Go on a phone, or a built app) is NOT affected —
  browsers enforce CORS, native fetch does not.

  Added: http://localhost:8081, http://127.0.0.1:8081 (Expo SDK 54 web
  port), plus :19006 variants (older Expo web port), for safety.

⚠️ IMPORTANT — THIS ONLY MATTERS IF RAILWAY HAS NO ALLOWED_ORIGINS VAR
  This array is only used when the ALLOWED_ORIGINS environment variable
  is NOT set on the running server (see src/index.ts line 465). Your
  mobile app talks to the PRODUCTION Railway API by default.

  ⚡ FASTEST FIX (no deploy, works immediately):
     Railway dashboard → contractnest-api service → Variables →
     if ALLOWED_ORIGINS exists, add these to the comma-separated value:
       http://localhost:8081,http://127.0.0.1:8081
     Railway restarts the service automatically — done in ~30 seconds.

  If ALLOWED_ORIGINS is NOT set on Railway (i.e. it's using this code's
  fallback array), then you need Steps 1-4 below (copy + commit + push
  + wait for Railway to redeploy from your main branch).
