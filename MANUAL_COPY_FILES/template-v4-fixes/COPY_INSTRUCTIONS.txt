BUGS FIXED IN v2
═══════════════════════════════════════════════════

BUG 1: Settings modal deactivate was broken
  CAUSE: PATCH handler used copy-on-write for ALL updates, including is_active toggle.
         This created a new inactive row AND hid the original (both invisible).
  FIX: Smart routing — metadata-only updates (is_active, is_public, etc.) now do
       in-place updates. Only content changes (name, blocks, tags) trigger versioning.

BUG 2: Template list didn't refresh after settings change
  CAUSE: Update mutation only invalidated catTemplateKeys.lists(), but the global
         templates page uses useCatSystemTemplates() keyed under catTemplateKeys.system().
  FIX: Now invalidates catTemplateKeys.all (covers all query types).

BUG 3: Queries failed if migration not yet run
  CAUSE: All GET handlers had hard .eq('is_latest', true) filter. If the column
         didn't exist in DB, entire query returned 400 error → zero templates shown.
  FIX: Try with is_latest first; if column-not-found error, retry without filter.
