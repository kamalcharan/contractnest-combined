WHAT WAS BROKEN: the contract wizard's Evidence Policy step selections
(evidence_policy_type, evidence_selected_forms) were silently discarded
whenever a contract was CREATED (via the wizard's initial save or VaNi
auto-accept). They only persisted if the contract was later EDITED and
saved, because update_contract_transaction already handled both fields
but create_contract_transaction's INSERT never referenced them at all.

Confirmed via direct inspection of both live RPC definitions:
  - contractnest-edge/supabase/functions/contracts/index.ts's
    handleCreate() spreads the full request body into the RPC payload
    with no whitelisting (`...body`) -- so the frontend was never the
    problem, same as the create_contract_transaction RPC itself was.
  - The wizard (contractnest-ui/.../ContractWizard/logic/mapper.ts) was
    already sending both fields correctly in the create payload.
  - The RPC's own INSERT column list was the only place the fields were
    dropped.

Files Changed:
  - contractnest-edge/supabase/migrations/contracts/072_persist_evidence_policy_on_create.sql
      Re-applies create_contract_transaction unchanged except for adding
      evidence_policy_type / evidence_selected_forms to the INSERT column
      list and VALUES (COALESCE to the same column defaults 'none' / '[]'
