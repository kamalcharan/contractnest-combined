FILE
  contractnest-ui/src/pages/settings/business-profile/smart-profile.tsx

WHAT CHANGED (and WHY)
  The page was MASKING AI/scrape failures by fabricating a profile:
    - generateFallbackDescription() -> "...professional organization committed
      to delivering high-quality services..." boilerplate  (REMOVED)
    - extractKeywordsFromText() + hardcoded ['Professional','Services',
      'Quality','Solutions'] keywords  (REMOVED)
    - both catch blocks advanced the wizard to 'enhanced' and showed a
      SUCCESS toast even though nothing real was produced.
  That fake data would flow into keywords -> clusters -> chips -> nomenclature
  -> template ranking = poisoned ICP. Now removed entirely.

  New behaviour (honest failure):
    • Enhance with AI (manual mode): only accepts a REAL AI result; empty is
      treated as failure. On failure -> ERROR toast, user STAYS on entry.
    • Analyze Website: only accepts REAL scraped content; empty = failure.
      On failure -> switches to MANUAL entry + ERROR toast (no fake profile).
    • NEW "Continue without AI" button (manual mode): proceeds with the
      user's OWN typed description verbatim (real input, empty keywords to be
      added manually) — this is the "switch to manual" path, now actually
      usable when AI/n8n is down.

  A profile now only ever contains REAL AI output or the user's OWN words —
  never invented text or keywords.
