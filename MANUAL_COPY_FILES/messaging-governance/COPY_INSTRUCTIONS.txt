BATCH: messaging-governance (2026-07-22)
═══════════════════════════════════════════════════
Owner decision: outbound messages are governed by RULES (Settings →
Automations, user-edited) AND CHANNELS (/integrations) — nothing goes out
unless BOTH allow it, EXCEPT three identity messages: password reset,
forgot password, and user invitations. Those always go out, but per the
owner's follow-up principle ("everything will go via jtd framework so
everything will be on audit-trail, if not its wrong") they must still be
visible in the n_jtd audit trail. See "IDENTITY-MESSAGE EXCEPTIONS" section
below for the three-function change that implements this.
