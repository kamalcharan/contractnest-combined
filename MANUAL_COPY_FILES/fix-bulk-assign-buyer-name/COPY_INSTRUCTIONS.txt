Branch: fix-bulk-assign-buyer-name

WHAT WAS WRONG: Bulk-assigning a template to multiple members gave every
resulting contract the same heading — "<Template> — New Client — <Duration>"
— instead of each member's real name. Direct/single assignment was correct.

ROOT CAUSE: BulkAssignDialog assembles the draft ONCE, before any member is
picked (buyer = null for the preview), so the server bakes the heading in
right then with the "New Client" fallback. Per-member submission overrode
buyerId/buyerName but never regenerated the baked-in contractName, so every
contract in the batch kept the placeholder heading.

Also separately noticed (and fixed): buyer names pulled straight from
contacts (often ALL CAPS from bulk imports) were going into contract
headings verbatim — "BBB 2026 Schedule — CHARAN KAMAL — 1 Year" — instead
of proper case.

THE FIX:
  contractnest-ui/src/components/contracts/vani/BulkAssignDialog.tsx
    - Per member, rebuild contractName by swapping the buyer segment of the
      baked-in heading for that member's real (title-cased) name, instead
      of leaving the "New Client" placeholder in every contract.
    - Added toTitleCase() so buyer names display as "Charan Kamal", not
      "CHARAN KAMAL".

  contractnest-api/src/services/contractComposerService.ts
    - assembleDraft() now title-cases the buyer name used in the generated
      contractName heading (affects direct/single assignment too, so this
      is fixed everywhere going forward, not just bulk).

DATA FIX (already applied directly to the live database — no file to
copy for this part): 26 existing contracts across 3 tenants had their
`name` corrected — the 9 BBB "New Client" placeholders now show the real
buyer name, and all buyer-name segments (BBB and others) were normalized
from ALL CAPS to proper case. This was a one-time SQL correction; nothing
further needed locally for it.

Files Changed:
  - contractnest-ui/src/components/contracts/vani/BulkAssignDialog.tsx
  - contractnest-api/src/services/contractComposerService.ts
