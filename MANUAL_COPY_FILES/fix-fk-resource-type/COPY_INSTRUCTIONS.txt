FIX: resource_type_id FK constraint violation on t_client_asset_registry
=====================================================================

ROOT CAUSE:
  resource_type_id in t_client_asset_registry has FK to m_catalog_resource_types(id)
  which stores VARCHAR(50) strings like 'equipment', 'asset'.
  The form was sending the tenant resource UUID (from t_catalog_resources) instead.

FIX:
  - resource_type_id = master catalog type string ('equipment'/'asset') -> satisfies FK
  - asset_type_id = tenant resource UUID -> used for sidebar filtering (no FK constraint)

SUBMODULES AFFECTED: contractnest-ui, contractnest-edge, contractnest-api
