FIX: Draft contracts being auto-activated + 502 errors
═══════════════════════════════════════════════════

Bug 1: Draft contracts created mid-wizard show as "Active"
Root Cause: saveDraftToApi used mapWizardToRequest() which includes
            acceptance_method: 'auto'. The DB RPC function checks
            "IF acceptance_method = 'auto' THEN status := 'active'"
            so drafts were immediately activated on creation.
Fix: saveDraftToApi now sends a minimal payload WITHOUT acceptance_method
     so the RPC defaults to draft status.

Bug 2: 502 Bad Gateway spam on PUT /api/contracts/<id>
Root Cause: Once a contract became active, the wizard's auto-save kept
            trying to PUT-update it, but the edge function rejects edits
            on non-draft contracts.
Fix: saveDraftToApi catches "not in draft status" errors and clears
     draftId to stop retrying. handleResumeDraft also guards against
     resuming non-draft contracts.
