KNOWN LIMITATION
─────────────────────────────────────────────────────
POST /api/seeds/tenant currently only runs 'sequences' in seedCategory().
The 'relationships' and 'eventStatuses' categories are registered in SeedRegistry
but seedRoutes.ts throws "Seed execution not implemented for category: X" for them.

This means Step 2 will partially succeed (sequences seeded, others skipped with errors).
The overall Step 2 status will still show done if sequences succeed (the API returns 207
for partial success). The UI shows "N records seeded" from totalInserted.

To fully seed relationships + eventStatuses, seedRoutes.ts needs:
  case 'relationships': return await seedRelationships(tenantId, isLive, authHeader);
  case 'eventStatuses': return await seedEventStatuses(tenantId, isLive, authHeader);
(This is a future backend task — not blocking for onboarding UX.)
