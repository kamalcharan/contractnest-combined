WHAT THIS IS
----------------------------------------------------------------
Phase 1 of the generic (tenant-agnostic) Group Sessions dashboard.
Four chair-side, tenant-scoped READ RPCs that power the dashboard
drill-down (Sessions -> Occurrences -> Roster -> Member).

A "group session" is defined GENERICALLY as any t_contracts row
flagged: metadata->>'group_session_owner' = 'true'
(the contract that OWNS the shared audience='group' schedule).
Roster reuses gs_membership_contract() from migration 017.
No tenant name is hardcoded - works for any customer.

RPCs added (all SECURITY DEFINER, RLS-on-no-policies pattern):
  1. gs_dash_sessions(p_tenant, p_is_live)   -> list sessions + aggregates
  2. gs_dash_occurrences(p_tenant, p_contract)-> occurrences + attendance
  3. gs_dash_roster(p_tenant, p_contract)     -> enumerate roster (NEW)
  4. gs_dash_member(p_tenant, p_member)       -> member attendance + dues
