WHAT THIS DELIVERS (owner design, 2026-07-02 sketch):
  1. TRUE PER-STEP PIPELINE — the composer is now 5 stateless endpoints:
     parse-intent (LLM) → resolve-buyer → shortlist → select-blocks (LLM) → assemble.
     Each canvas card corresponds to exactly one completed call. No simulated
     progress, ever.
  2. THE VANI CANVAS — left: VaNi builder box (live narration, real elapsed
     seconds); right: step cards that ANIMATE OUT OF THE VANI BOX along a
     dotted connector, one per completed step. Interactive cards pause the
     pipeline for user action (buyer disambiguation / continue-without-buyer),
     with per-step retry on errors.
  3. SINGLE-VIEW REVIEW & FINALIZE — no wizard walk needed: the canvas flows
     into the contract view (reuses ReviewSendStep's paper canvas) with
     Approve & Send. "Edit in wizard" remains the fallback (vaniPrefill).
  4. ONE WRITE PATH — finalize goes through useContractSubmission which uses
     the wizard's own exported mapWizardToRequest + the existing
     createContract/updateStatus/sendNotification operations. Totals are now
     computed server-side in /assemble so the review can send without the
     wizard's billing step.
     · Auto-accept drafts route to the wizard (the pre-payment dialog lives
       there); payment/signoff acceptance finalize directly from the canvas.
  5. Send is blocked (with explicit reasons) until hard requirements are met:
     buyer selected, blocks present, coverage picked for asset-group contracts
     — same rules the wizard's gated steps enforce.
