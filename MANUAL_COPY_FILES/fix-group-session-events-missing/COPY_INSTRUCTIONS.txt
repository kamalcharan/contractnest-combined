Branch: fix-group-session-events-missing

WHAT WAS WRONG: found while auditing all LIVE contracts for a reported
start-date issue. Every contract built from a template containing a Group
Session block (e.g. BBB's "Saturday Cadence", biweekly, 25 occurrences/year)
had ZERO scheduled occurrence events — not wrong dates, completely absent.
Confirmed across all 27 live "BBB 2026 Schedule" contracts.

ROOT CAUSE: the event derivation engine (contractEventsDerivationService.ts
on the API, and its frontend parity copy contractEvents.ts) decides whether
a block needs scheduled service events by calling categoryHasPricing(),
which only recognizes categoryId 'service' and 'spare'. A Group Session
block uses categoryId 'session' and is often complimentary (price 0 on
purpose — attendance tracking, not billing) — so it has no pricing step in
the wizard, categoryHasPricing() returned false, and the block was silently
skipped. No error anywhere; the schedule just never existed.

THE FIX: split the concern. categoryHasPricing() still gates BILLING events
(correct — a 0-price session block shouldn't generate a billing event).
Service-event generation now checks a new, separate
categoryNeedsServiceEvents() (service + session) instead, so Group Session
blocks get their occurrence schedule regardless of price.

DATA FIX (already applied directly to the live database — no file to copy
for this part): backfilled the missing 25-occurrence Saturday-biweekly
schedule for all 27 existing live "BBB 2026 Schedule" contracts, anchored
to each contract's (corrected) start date. 675 events inserted total.

Also fixed in this same pass (separate issue, same audit): 8 of the 27 BBB
contracts had been bulk-assigned with today's date as the contract start
instead of the intended fiscal year (1 Apr 2026 – 1 Apr 2027) — corrected
the contract dates and re-aligned their billing event schedule to match the
other 19 (already-issued/paid invoices and receipts were left untouched,
since those reflect real money that was actually collected — only the
event schedule *labels* were corrected).

Files Changed:
