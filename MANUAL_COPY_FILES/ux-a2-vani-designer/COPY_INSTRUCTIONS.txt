WHAT THIS IS
  Adds the admin-side agent panel (prototype "View 4") to the Global
  Template Designer as a right slide-in drawer. Admin clicks "Design with
  VaNi" -> context strip (industry/asset/nomenclature) -> 4 quick actions
  (Generate Full / Suggest Slots / Auto-Fill / Review) -> 7-step streaming
  -> result. Slot results have per-slot accept/reject; Apply writes them to
  the wizard (recipeSlots) and, on Save, into t_cat_templates.settings.recipeSlots.

  RECIPE = SLOTS (activity + asset + cadence + compliance), NOT block_ids.
  This is the slot-hydration model: tenant-time seeding hydrates slots against
  the tenant's own catalog blocks. The existing block_id path is untouched.

  NOTE: the generator is a DETERMINISTIC STUB in this batch (no LLM) so the UX
  is fully clickable/testable. Batch C swaps in /api/agent/recipe/* + edge; the
  panel/hook public surface stays identical.

  contractnest-api / contractnest-edge / ClaudeDocumentation /
  ContractNest-Mobile : NOT affected.  FamilyKnows: N/A (removed).
