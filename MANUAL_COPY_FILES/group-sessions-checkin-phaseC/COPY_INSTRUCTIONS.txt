WHAT THIS ADDS
----------------------------------------------------------------
Check-in now works off the BLOCK schedule, so scanning the QR finds today's
occurrence and writes attendance back — lighting up present counts,
attendance %, and QR-ready on the dashboard.

  * Token per BLOCK (+ environment): gs_ensure_block_token. The dashboard
    "Open check-in" / "Check-in" mints it and opens /checkin/<token>.
  * gs_resolve_checkin / gs_lookup_member / gs_member_history / gs_submit_checkin
    / gs_checkin_form are now BLOCK-aware (legacy contract tokens still work).
  * Attendance rows link to the schedule occurrence (t_session_attendance
    .schedule_occurrence_id + .source_block_id), so:
      - Occurrences tab shows "· N present" on past/held rows
      - roster "Attended" counts real check-ins
      - gs_dash_sessions returns qr_ready + attendance_pct

The public member check-in page (SessionCheckinPage) is UNCHANGED — it works
transparently through the now-block-aware token RPCs.

Schema notes (in migration 022): t_group_session_tokens.contract_id and
t_session_attendance.session_contract_id are made NULLABLE (block tokens /
block attendance have no single contract); + new columns/indexes.
