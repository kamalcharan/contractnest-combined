  1. contractnest-api/src/services/contractEventsDerivationService.ts
     Server-side derivation of the contract forward calendar (computed_events).

     backend/agent draft a contract without the browser.

  2. contractnest-api/src/services/vaniLLMClient.ts
     Gateway for all VaNi LLM calls — self-hosted Qwen (OpenAI-compatible),
     env-swappable model, /no_think enforced, JSON-repair parsing,
     graceful degradation when the LLM VPS is down.

  3. contractnest-api/src/services/vaniInteractionLogger.ts
     Logs every LLM call to vn_interaction_log via SECURITY DEFINER RPCs
     (auditService pattern, anon key). Fire-and-forget — logging failures
     never break the user flow. Includes recordFeedback() for
     was_edited / was_accepted / rating (the fine-tuning gold signals).

  4. contractnest-api/src/__tests__/contractEventsDerivationParity.ts
     Parity test: backend derivation vs verbatim UI algorithm, 6 scenarios.
     PASSED in dev environment: all byte-identical.

  5. contractnest-api/src/__tests__/vaniLLMSmoke.ts
     End-to-end smoke: Qwen call -> JSON parse -> interaction logged ->
     feedback recorded. Run after setting env + applying the migration.

  6. contractnest-edge/supabase/migrations/vani-agent/001_create_vn_interaction_log.sql
     vn_interaction_log table (LLM Strategy v1.0 schema) + insert/feedback
     RPCs. New objects only — touches NO existing table, grant, or policy.
     Table is RLS-locked with zero policies; the RPCs are the only write path.
