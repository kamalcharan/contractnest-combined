COPY INSTRUCTIONS — kt-stepwise-gen-fix
Fix: Stepwise saves were wiping all unrelated tables
=====================================================

ROOT CAUSE:
  The edge function's save endpoint calls wipeLiveData() which deletes ALL tables
  (variants, spare_parts, checkpoints, overlays) before inserting.
  Sending only spare_parts data would wipe variants and checkpoints.

FIX:
  1. Edge function (wipeLiveData): added save_mode parameter.
     When save_mode is 'variants'/'spare_parts'/'checkpoints'/'overlays',
     only that scope is wiped. Full wipe only when save_mode is absent (existing behavior preserved).

  2. UI hooks (useKnowledgeTree.ts): each stepwise save now sends save_mode:
     - useKTGenerateVariants   → save_mode: 'variants'
     - useKTGenerateSpareParts → save_mode: 'spare_parts'
     - useKTGenerateCheckpoints→ save_mode: 'checkpoints'
     - useSaveContextOverlays  → save_mode: 'overlays'

  Manual "Save Changes" (useKnowledgeTreeSave) sends no save_mode → full wipe as before.

FILES CHANGED:
  contractnest-edge:
  1. supabase/functions/knowledge-tree/index.ts  [MODIFIED]
     — wipeLiveData: added save_mode parameter with selective wipe paths
     — saveKnowledgeTree: destructures save_mode from body, passes to wipeLiveData

  contractnest-ui:
  2. src/hooks/queries/useKnowledgeTree.ts       [MODIFIED]
     — useKTGenerateVariants: adds save_mode:'variants' to save payload
     — useKTGenerateSpareParts: adds save_mode:'spare_parts' to save payload
     — useKTGenerateCheckpoints: adds save_mode:'checkpoints' to save payload
     — useSaveContextOverlays: adds save_mode:'overlays' to save payload
