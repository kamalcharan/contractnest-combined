HOW IT WORKS
─────────────────────────────────────────────────────
POST /api/seed/tenant/industry-confirmed
  Headers: Authorization: Bearer <token>, x-tenant-id: <uuid>
  Body:    { "industryId": "lifts_elevators", "businessType": "seller" }

  Step 1  Idempotency guard — returns cached result if already run
  Step 2  Clones global KT blocks (m_cat_blocks WHERE tenant_id IS NULL
          AND tags contains industryId) → tenant-scoped copies with
          is_live=false, is_seed=true
  Step 3  Buyer/both only: reads m_facility_hierarchy_templates for
          industryId, inserts one t_client_asset_registry placeholder
          per hierarchy level (ownership_type='self', is_live=false,
          specifications.is_placeholder=true)
  Step 4  Seeds sequences for live + test environments (non-fatal if
          already seeded)
  Step 5  Stores t_idempotency_keys record (1-year TTL)
