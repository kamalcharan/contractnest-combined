Branch: sprint2-auto-generate-kt-forms
(Sprint 2/7 pilot — auto-generate SmartForms as part of KT generation +
tenant onboarding, no manual "export" click required)

WHAT THIS DOES:
1. Right after an admin completes the KT "Generate Service Names" step
   (the point at which service_name is finally persisted, making
   checkpoint grouping resolvable — catalog_name is already there from
   the earlier, automatic Cycles step), the system now AUTOMATICALLY:
   composes one small SmartForm per service group (same grouping key
   ktCatBlockMapperService already uses: catalog_name || service_name),
   creates it, and registers it in a new master mapping table
   (m_kt_service_form_map) — keyed by resource_template_id + service_name,
   independent of any tenant.
2. Tenant onboarding's existing bulk-seed step (cat-blocks/bulk edge
   route, unchanged trigger point) now looks up that mapping table and
   stamps form_template_id onto each new tenant's m_cat_blocks row
   automatically at seed time. Every tenant who onboards with a given
   equipment from now on gets pre-armed service blocks — no admin action
   per tenant, ever.

This makes forms a genuine peer to pricing/blocks in the onboarding
sequence, and removes the need for the "Generate & Attach" per-block
button (from the sprint2-arm-catalog-studio batch, not yet copied into
your local tree) as the PRIMARY path — that button still exists and still
works, but is now a manual override/regenerate tool for one already-
seeded tenant's block, not the only way forms get attached.

  - src/services/ktCatBlockMapperService.ts
      New public method getServiceCheckpointGroups() — same grouping key

      useAutoComposeForm.ts (composeBlockForm + its helpers) — pure data
      transformation, no React dependency, so it runs identically
      server-side. Skips any service group already in the mapping table
      (won't regenerate/duplicate on repeated "Generate Service Names"
      runs).
  - src/controllers/knowledgeTreeController.ts
      New generateForms handler — POST /api/knowledge-tree/generate-forms.
      No LLM call; purely deterministic from already-generated data.

Production Checklist:

     sequentially per service group inside generateFormsForTemplate;
