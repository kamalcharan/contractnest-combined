UNIFIED BILLING CYCLE - Validation on Continue
================================================

Changes: Removed the unified cycle selector bar. All billing cycle selectors
remain visible on every block. When user clicks Continue from the Add Service
Blocks step and billingCycleType is 'unified', validation checks that ALL
pricing blocks have the same billing cycle. If mismatched, a toast error is
shown and navigation is blocked.

   - Removed hideBillingCycle from block card props
   - Cleaned up unused imports
