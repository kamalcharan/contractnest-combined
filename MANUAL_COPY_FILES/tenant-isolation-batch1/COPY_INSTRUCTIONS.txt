---------------------------------------------------------------

bypass on 11 tables that currently have RLS DISABLED while holding
tenant_id data (any authenticated user could read all tenants' rows).

---------------------------------------------------------------

HOW IT WAS VALIDATED (no prod changes persisted)
---------------------------------------------------------------
  Applied inside a BEGIN ... ROLLBACK transaction on the live DB and
  impersonated two real cross-tenant users via SET ROLE authenticated +
  request.jwt.claims:
      total rows (RLS off) ......... 146  (all tenants visible — the bug)
      User A visible after batch1 ...   5  (own tenant only)  PASS
      User B visible after batch1 ...   3  (own tenant only)  PASS
  Rollback verified: RLS still off, 0 batch1 policies left, scratch table gone.

  ROLLBACK: run 001_..._DOWN.sql the same way.
