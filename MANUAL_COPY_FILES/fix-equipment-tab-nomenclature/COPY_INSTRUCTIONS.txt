FIX: Equipment tab showing "Equipment" instead of "Facility" for facility contracts
================================================================================

ROOT CAUSE:
The contract detail page used `useNomenclatureTypes` hook which made a raw fetch()
call to `/api/functions/product-masterdata` — a path that doesn't exist on the
Express API server. This silently failed, so nomenclature data was never loaded,
and the tab always defaulted to "Equipment".

FIX:
Replaced `useNomenclatureTypes` with `useGlobalMasterData('cat_contract_nomenclature')`
— the exact same hook the contract wizard already uses successfully. This uses the
correct API path `/api/product-masterdata/global` through the axios `api` instance.

SUBMODULES AFFECTED:
- contractnest-ui
