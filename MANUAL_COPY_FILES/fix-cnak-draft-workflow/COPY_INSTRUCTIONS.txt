WHAT THIS FIX DOES:
═══════════════════════════════════════════════════════════════

Root Cause: Migration 050 changed CNAK format from 'CNAK-' + 6 hex chars
(11 chars total) to XXXX-XXXX-XXXX (14 chars total), which exceeds the
VARCHAR(12) column limit on t_contracts.global_access_id.

Fix 1 (create_contract_transaction):
  - Reverts CNAK format back to CNAK-XXXXXX (11 chars, fits VARCHAR(12))
  - For drafts: sets v_cnak = NULL, skips t_contract_access INSERT
  - For non-drafts (auto-accept): generates CNAK as before

Fix 2 (update_contract_status):
  - New STEP 2.5: when transitioning FROM draft to non-draft (except cancelled)
    AND global_access_id IS NULL, generates CNAK-XXXXXX and creates
    t_contract_access row
  - All existing behavior preserved for non-draft transitions
