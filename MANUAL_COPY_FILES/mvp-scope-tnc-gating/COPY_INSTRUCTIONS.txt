───────────────────────────────────────────────────────────────────
WHAT THIS BATCH DOES
───────────────────────────────────────────────────────────────────

1) SINGLETON TERMS & CONDITIONS (Catalog Studio)
   - Text block category renamed to intent: "Terms & Conditions
     (one per business)" and flagged singleton.
   - Creating a second text block is intercepted: the wizard opens the
     tenant's EXISTING T&C in edit mode instead (info toast explains).

2) COMING-SOON GATING (Catalog Studio)
   - Video / Image / Document block types show a "Coming soon" badge,
     dimmed and non-clickable. Post-MVP smart canvas replaces them.

3) T&C IN VANI ONBOARDING (new step)
   - New step at /onboarding/terms-conditions, between pricing-review
     and equipment-confirm (both persona) / lov-setup (seller persona).
   - VaNi presents editable boilerplate in RichTextEditor; found
     existing T&C is loaded instead of the boilerplate (reused tenants).
   - Saving creates/updates the singleton text block through the same
     catalog mutations Catalog Studio uses (toasts included).
   - Skippable — "Skip for now" defers to Catalog Studio (lazy seed).

4) AUTO-INCLUDE T&C IN EVERY CONTRACT & TEMPLATE (Contract Wizard)
   - On wizard open (contracts AND templates), the tenant's T&C text
     block is appended automatically at ₹0 (no billing, no events);
     re-appended if a template/draft predating this rule lacks it.
   - RFQ mode: auto-included T&C is stripped (RFQs carry no doc terms).
   - Its content renders as the Terms section of the contract document.

5) FLYBY RESTRICTED TO SERVICES (Contract Wizard)
   - Ad-hoc FlyBy block creation in Add Service Blocks now offers ONLY
     the Service type (text/document FlyBys could break the document
     structure).

6) ON-SCREEN REVIEW = CONTRACT DOCUMENT (owner decision #4: "same")
   - Wizard Review & Send: the old paper preview is replaced by the
     professional ContractDocument (exactly what the PDF exports).
     Self/Client toggle retired with the paper; RFQ keeps old preview.
   - Buyer review page (/contract-review): same — document renders
     visibly; legacy paper kept only as fallback if document data
     can't be assembled. PDF export captures the visible document.
