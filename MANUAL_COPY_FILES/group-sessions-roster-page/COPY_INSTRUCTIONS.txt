WHAT CHANGED (per your last message)
────────────────────
1. Group view KPI strip — the "Dues due" tile is replaced by a
   "Roster View" tile: same size/position as the other 3 stat tiles,
   but visually distinct (solid brand-primary background instead of
   the neutral bordered style) so it reads as a button, not a stat.
   Clicking it opens the new Roster page for that session.

2. The Roster table that used to sit at the bottom of the Group view's
   left column is gone — Roster only lives on its own page now.

3. New Roster page (Group Sessions → a session → Roster View):
   - 4 stat tiles: Members, Average attendance (till now, computed
     from each member's own attended/overall — no hardcoded numbers),
     Over cap (count of members over EITHER their own no-show or
     substitute cap — each member's cap comes from their own signed
     contract snapshot, same source as everywhere else in this
     feature — nothing hardcoded), Dues due.
   - A search box, filters the list by member name as you type
     (client-side — the roster is already fully loaded, no new
     network calls).
   - A list view styled like the Contacts list page (rounded row
     cards, not a dense table) — one row per member: avatar, name,
     contract, Attended/Missed/Subbed, cap status pill, dues pill,
     "View contract" link (same as before — /contracts/:id?tab=sessions).

WHY THIS IS SAFE / MINIMAL
────────────────────
- No DB or API changes — same gs_dash_roster data, just fetched when
  view==='roster' too (previously only fetched for view==='group').
- No new route — 'roster' is a new internal view state alongside the
  existing overview/group/occurrence pattern already used throughout
  this page, not a new URL.
