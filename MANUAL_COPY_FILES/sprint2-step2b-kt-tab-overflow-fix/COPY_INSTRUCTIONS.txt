(Bug fix found while verifying Sprint 2/7 Step 2)

Files Changed:
  - contractnest-ui/src/pages/service-contracts/templates/admin/knowledge-tree/KnowledgeTreeDetail.tsx
      The Knowledge Tree detail page's tab bar (Variants / Spare Parts /
      Checkpoints / Service Cycles / Services / Overlays / Form Preview)
      had no overflow handling. The page layout is a 2-column CSS grid
      (`1fr 320px`) with `min-w-0` on the left column, so on narrower
      windows/zoom the 7-tab row overflowed past the left column boundary
      and visually spilled underneath the "VaNi Intelligence" card in the
      right column (which paints on top since it's later in the DOM) —
      making the "Form Preview" tab appear to have vanished.

      Fix: tab bar container now scrolls horizontally
      (overflowX: 'auto', flexWrap: 'nowrap') and each tab button no
      longer wraps its label (whiteSpace: 'nowrap', flexShrink: 0), so
      on narrow screens you get a scrollable tab strip instead of visual
      clipping/overlap. No tabs added/removed/reordered.

Submodules Affected: contractnest-ui
